# 发布订阅模式 (Pub-Sub Pattern)

发布订阅模式核心是解耦发布者和订阅者，通过主题/事件通道实现异步通信。

## 一、观察者模式实现

```java
// 1. 定义订阅主题接口
interface Subject {
    void attach(Observer observer);
    void detach(Observer observer);
    void notifyObservers(Event event);
}

// 2. 定义观察者接口
interface Observer {
    void update(Event event);
}

// 3. 事件定义
class Event {
    private String type;
    private String data;

    Event(String type, String data) {
        this.type = type;
        this.data = data;
    }

    String getType() { return type; }
    String getData() { return data; }
}

// 4. 具体主题实现
class NewsAgency implements Subject {
    private List<Observer> observers = new ArrayList<>();
    private String news;

    void attach(Observer observer) {
        observers.add(observer);
    }

    void detach(Observer observer) {
        observers.remove(observer);
    }

    void publishNews(String news) {
        this.news = news;
        notifyObservers(new Event("news", news));
    }

    void notifyObservers(Event event) {
        observers.forEach(o -> o.update(event));
    }
}

// 5. 具体观察者实现
class Subscriber implements Observer {
    private String name;

    Subscriber(String name) { this.name = name; }

    void update(Event event) {
        System.out.println(name + " 收到: " + event.getData());
    }
}

// 6. 使用示例
Subject agency = new NewsAgency();
agency.attach(new Subscriber("张三"));
agency.attach(new Subscriber("李四"));
agency.publishNews("Java 23 发布了");
```

## 二、消息代理实现（异步解耦）

```java
// 消息代理
class MessageBroker {
    Map<String, List<Consumer>> topics = new HashMap<>();

    void subscribe(String topic, Consumer consumer) {
        topics.computeIfAbsent(topic, k -> new ArrayList<>()).add(consumer);
    }

    void unsubscribe(String topic, Consumer consumer) {
        topics.getOrDefault(topic, List.of()).remove(consumer);
    }

    void publish(String topic, Object message) {
        // 实际项目中这里可改为异步队列
        topics.getOrDefault(topic, List.of()).forEach(c -> c.accept(message));
    }
}

// 使用示例
MessageBroker broker = new MessageBroker();

Consumer subscriberA = msg -> System.out.println("订阅者A: " + msg);
Consumer subscriberB = msg -> System.out.println("订阅者B: " + msg);

broker.subscribe("java", subscriberA);
broker.subscribe("java", subscriberB);
broker.subscribe("python", msg -> System.out.println("Python爱好者: " + msg));

broker.publish("java", "JDK 23 新特性");
// 输出:
// 订阅者A: JDK 23 新特性
// 订阅者B: JDK 23 新特性
```

## 三、两种风格对比

| 维度 | Observer 模式 | Pub-Sub 模式 |
|------|--------------|--------------|
| 耦合度 | 发布者和观察者直接通信 | 通过消息代理解耦 |
| 通信方式 | 同步调用 | 支持异步 |
| 通信关系 | 一对多 | 多对多 |
| 典型场景 | GUI 事件处理 | 消息队列、系统集成 |

## 四、Java 标准库支持

```java
// 使用 Observable/Observer (Java 9 已废弃，仅示例)
class NewsPublisher extends Observable {
    void publish(String news) {
        setChanged();
        notifyObservers(new Event("news", news));
    }
}

// 使用 PropertyChangeListener
class Publisher {
    private final PropertyChangeSupport support = new PropertyChangeSupport(this);

    void addPropertyChangeListener(PropertyChangeListener listener) {
        support.addPropertyChangeListener(listener);
    }

    void notify(String msg) {
        support.firePropertyChange("event", null, msg);
    }
}
```

## 五、关键设计原则

1. **面向接口编程**：Subject 和 Observer 都应该是接口
2. **注意内存泄漏**：订阅后要及时取消订阅
3. **异常处理**：观察者更新时抛异常不能影响其他观察者
4. **线程安全**：多线程环境下需考虑同步问题
